About This Project

🎯 What Problem This Solves

Most consumer goods companies manage their supply chain planning through disconnected Excel files — one per region, each with different column names, currencies, and formats. Nobody has a single source of truth.


This project simulates exactly that problem and solves it end to end: from messy regional spreadsheets to a unified, automated, forecast-driven planning hub — built in one week.

🛠 Technology Stack

🐍

Python (pandas, numpy)

Data cleaning, standardisation, pipeline automation

🐘

PostgreSQL

Production-grade shared planning database with views

📊

Statsmodels (ETS)

Holt-Winters exponential smoothing demand forecast

🌐

HTML + Chart.js

Self-contained dashboard, hosted on GitHub Pages

📋 What Was Built (Day by Day)

  • Day 1 — Generated realistic messy regional Excel files (AU, EU, USA)
  • Day 2 — Data cleaning pipeline → PostgreSQL database with 3 views
  • Day 3 — Statistical demand forecasting (6 months, all SKU x Region)
  • Day 4 — Inventory risk engine: coverage, risk flags, reorder logic
  • Day 5 — This dashboard: self-contained HTML, GitHub Pages deployed

💡 Why AI Alone Can't Do This

The CEO concern: "AI will just consolidate the Excel files."


The reality: AI needs clean, structured, consistent data to be useful. Without someone to standardise column names across regions, resolve currency differences, fix broken categories, and define what "inventory risk" means for this business — AI produces garbage.


This role builds the foundation that makes AI useful. Without it, the AI investment fails.
